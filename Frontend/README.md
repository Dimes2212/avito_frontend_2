# Frontend Trainee Assignment (Spring 2026)

Тестовое задание: личный кабинет продавца с AI-ассистентом для улучшения объявления.

## Что реализовано

Приложение содержит 3 основные страницы:

1. `/ads` — список объявлений
2. `/ads/:id` — просмотр карточки объявления
3. `/ads/:id/edit` — редактирование объявления

### Функционал `/ads`

- Заголовок «Мои объявления» + количество объявлений
- Поиск по названию
- Сортировка:
  - по названию (А-Я / Я-А)
  - по новизне (новые / старые)
  - по цене (дешевле / дороже)
- Боковые фильтры:
  - категории
  - «Только требующие доработок»
  - «Сбросить фильтры»
- Пагинация (10 карточек в grid-режиме)
- Переход в карточку объявления по клику
- Переключение видов отображения (grid/list)
- Состояния загрузки и ошибки

### Функционал `/ads/:id`

- Название, цена, даты публикации/редактирования
- Плейсхолдер изображения
- Характеристики по категории
- Описание (fallback: «Отсутствует»)
- Блок «Требуются доработки» при незаполненных полях
- Кнопка «Редактировать»
- Кнопка «Отменить» (возврат к `/ads`)
- Состояния загрузки и ошибки

### Функционал `/ads/:id/edit`

- Редактирование:
  - категория (select)
  - название (обязательное)
  - цена (обязательное, number)
  - характеристики по категории (необязательные)
  - описание со счетчиком символов (до 1000)
- Валидация обязательных полей
- Сохранение через `PUT /items/:id`
- После успешного сохранения: редирект на `/ads/:id`
- «Отменить»: возврат на `/ads/:id` без сохранения
- Состояния загрузки/ошибки + snackbar
- Черновик формы в `localStorage` (восстановление после перезагрузки)

### AI-функции

На странице редактирования:

- «Придумать описание» / «Улучшить описание»
- «Узнать рыночную цену»
- После ответа AI:
  - «Применить» (подставляет ответ в поле)
  - «Закрыть»
  - «Повторить запрос»

AI реализован через локальный Ollama (`POST /api/generate`) на бэкенде.

## Установка и запуск

## 1) Backend

```bash
cd server
npm i
```

Создай `.env` (пример уже есть в `server/.env.example`):

```env
OLLAMA_URL=http://127.0.0.1:11434
OLLAMA_MODEL=qwen2.5:3b
```

Запуск:

```bash
npm start
```

Сервер поднимется на `http://localhost:8080`.

## 2) Frontend

```bash
cd frontend
npm i
npm run dev
```

Фронт поднимется на `http://localhost:5173`.

---

## Настройка LLM (Ollama)

Установить Ollama:

- https://ollama.com/download

Скачать модель (пример):

```bash
ollama pull qwen2.5:3b
```

Запустить Ollama:

```bash
ollama serve
```

Бэкенд обращается к Ollama напрямую:

- `POST http://localhost:11434/api/generate`

## Запуск через Docker Compose

В проекте есть:

- `docker-compose.yml` (корень)
- `frontend/Dockerfile`
- `server/Dockerfile`

Порядок запуска:

1. Убедиться, что Docker Desktop запущен.
2. Из корня проекта выполнить:

```bash
docker compose build
docker compose up
```

После запуска доступны:

- Frontend: `http://localhost:5173`
- Backend: `http://localhost:8080`
- Ollama API: `http://localhost:11434`

Остановка:

```bash
docker compose down
```

Запуск в фоне:

```bash
docker compose up -d
```

Просмотр логов:

```bash
docker compose logs -f
```

Если нужно пересобрать образы после изменений:

```bash
docker compose down
docker compose build --no-cache
docker compose up
```

## API, используемое клиентом

- `GET /items` — список объявлений
- `GET /items/:id` — объявление по id
- `PUT /items/:id` — сохранение изменений
- `POST /ai/description` — AI-описание (вызов Ollama под капотом)
- `POST /ai/price` — AI-оценка цены (вызов Ollama под капотом)

## Принятые решения

1. Для AI вызовы идут через backend:
   - фронт не знает деталей LLM-интеграции
   - единая точка обработки ошибок и prompt-логики

2. Для UX редактирования добавлен черновик в `localStorage`:
   - восстановление полей после перезагрузки страницы
   - удаление черновика после успешного сохранения

3. Для `/ads` используется client-side обработка списка:
   - поиск/фильтрация/сортировка/пагинация поверх загруженных данных
   - это упрощает и ускоряет работу интерфейса

4. Backend из задания изменялся минимально:
   - в ответ `GET /items` добавлено поле `id` (для корректной навигации и ключей карточек на фронте)
   - добавлены AI-эндпоинты `POST /ai/description` и `POST /ai/price` (обертка над Ollama API)
   - добавлена базовая CORS/OPTIONS-обработка для корректной работы запросов из браузера
   - подключена загрузка переменных окружения из `server/.env` (для `OLLAMA_URL`, `OLLAMA_MODEL`)
   - для корректной работы в Docker сервер слушает `0.0.0.0` (через `HOST`, по умолчанию `0.0.0.0`)

5. Контейнеризация:
   - добавлены `docker-compose.yml`, `frontend/Dockerfile`, `server/Dockerfile`
   - `docker compose up` поднимает `frontend + server + ollama` в одной сети
   - отдельный сервис `ollama_init` автоматически скачивает модель `qwen2.5:3b`
